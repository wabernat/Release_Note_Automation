RING 8 consists of two distinct components: the RING and the S3 Connector. Because these components are on separate development paths, each RING 8 release may embed different version increments of each component. The initial release of RING 8, for example, comprised RING v8.0.0.0 and S3 Connector v7.5.0.0.

RING 8 Product Organization

RING 8 encompasses:

  • RING Core, Installation, Supervision, and Monitoring
  • Object Connectors
  • Scale-Out Filesystem (SOFS) Connector Options
  • Email and Messaging Connectors

RING Core, Installation, Supervision, and Monitoring

This includes:

  • All core RING features, including the full scale-out architecture; self-healing, flexible data protection policies with replication; and Advanced Resiliency Configuration (ARC) erasure coding. It also includes RING core improvements that enhance performance, scalability, and maintainability.
  • The RING Installer and its set of pre-install and post-install check tools. Its hardware inventory streamlines typical RING installations on known x86 hardware.
  • The RingSH command-line interface, the Supervisor UI, and the set of RESTFul APIs for cloud or local RING management and monitoring.
  • Support for single-site and multiple (2 or 3)-site geo-distributed deployments for file and object connectors.

Scale-Out Filesystem (SOFS) Connector Options

FUSE (sfused) connector Local Linux host file system access to RING storage services.
NFSv3 connector Standard NFSv3 network file protocol mountpoint access to RING storage services.
NFS Ganesha connector New NFS connector supporting the NFSv3 and NFSv4 protocols, IPv6 support, Kerberos authentication, and POSIX ACL.
SMB 2.0/3.0 connector SMB 2.0 and SMB 3.0 (subset)-compliant access over shares to RING storage services.
CDMI REST connector SNIA CDMI 1.0.1-compliant REST interface, with access by path semantics.
Multi-tenant & Kerberos option Optional Kerberos KDC/Microsoft Active Directory server-based authentication support for NFSv3 and SMB 2.0/3.0 connectors.

Email and Messaging Connectors

Zimbra 8.0.3 and later and Zimbra X For support of the Zimbra Collaboration solution version 8 (http://www.zimbra.com/) over the sproxyd REST connector.
Dovecot PRO Supported via the RING sproxyd connector for single and multi-geo deployments. Existing single-site deployments of Dovecot PRO with CDMI continue to be supported; however, support is deprecated for new deployments.
Openwave Messaging Mx 8.4 and later For support of the Openwave Mx messaging solution suites over the RING RS2 Lite connector.
